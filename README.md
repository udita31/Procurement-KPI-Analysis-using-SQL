# Procurement-KPI-Analysis-using-SQL
Analyze procurement data to evaluate supplier performance, delivery efficiency, cost savings, quality defects, and compliance risks using advanced SQL techniques.
Objective:
Analyze procurement data to evaluate supplier performance, delivery efficiency, cost savings, quality defects, and compliance risks using advanced SQL techniques.

# Dataset Fields:
- PO_ID 
- Supplier
 - Order_Date
 - Delivery_Date
 - Item_Category
 -  Order_Status
 -   Quantity
 -   Unit_Price
 -   Negotiated_Price
 -   Defective_Units
 -   Compliance

# Key Business Problems Solved:

- Supplier delivery performance and delay analysis
- Defect rate and quality risk assessment
- Cost savings through negotiated pricing
- Monthly procurement spend trend analysis
- Supplier dependency and compliance risk evaluation









# SQL Techniques Used:

- Common Table Expressions (CTEs)
- Window Functions (RANK, LAG)
- Conditional Aggregation
- Time-Series Analysis
- KPI Calculations & Percentages









# Outcome:

- Built supplier scorecards combining delivery, quality, and cost KPIs
- Identified high-risk suppliers based on defects and non-compliance
- Highlighted procurement cost-saving opportunities
  


# Tools:
- MySQL 8.0
- MySQL Workbench
- CSV Exports
